# План внедрения Ежедневной Статистики

## Идея
Добавить переключатель "Режим" (День / Неделя) на Дашборд и в форму добавления. Это позволит вести ежедневный трекинг, который не смешивается с основным недельным/месячным отчетом.

## Архитектура
1.  **База данных (`types.ts`)**:
    *   Добавляем поле `days` в объект сотрудника (аналогично `weeks`).
    *   Структура: `days: { "2024-02-01": { kpi: 100, chats: 50... } }`.

2.  **Ввод данных (`AddEmployee.tsx`)**:
    *   Добавляем переключатель: **"Тип отчета: Неделя / День"**.
    *   Если выбран "День" — скрываем выбор недели, показываем выбор **одной даты**.
    *   Сохраняем данные в новую ветку `days` в Firebase.

3.  **Просмотр (`Dashboard.tsx`)**:
    *   Добавляем переключатель сверху: **"Обзор: Период / Ежедневный"**.
    *   В режиме "Ежедневный":
        *   Появляется календарь для выбора конкретного дня.
        *   Статистика строится только на основе данных этого дня.
        *   Если заполнила только Команда 1 — показываем только её.
        *   Если заполнили обе — показываем сравнение (как сейчас).

## Преимущества
*   Не ломает текущий функционал (он останется по умолчанию).
*   Позволяет сравнивать команды "онлайн" за сегодня/вчера.
*   Гибкость: можно заполнять хоть каждый день, хоть раз в неделю — системы независимы.

## UI Изменения
*   Новые табы (в стиле "светящегося стекла" taxi-yellow).
*   Календарь для выбора дня.
