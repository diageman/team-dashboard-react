# План интеграции Firebase (Общая база данных)

Нужно перевести приложение с `localStorage` (данные только у вас) на `Firestore` (данные общие для всех).

## 1. Настройка подключения (src/lib/firebase.ts)
Создадим файл конфигурации с вашими ключами. Этот файл будет отвечать за связь с серверами Google.

## 2. Переписывание логики хранения (src/lib/storage.ts)
Сейчас функции `getEmployees` и `saveEmployeeData` работают синхронно (мгновенно читают из памяти браузера).
База данных работает асинхронно (нужно время на запрос к интернету).

*   Переделаем функции на `async/await`.
*   `getEmployees` -> `fetchEmployees` (получение коллекции из базы).
*   `saveEmployeeData` -> `addEmployee` / `updateEmployee` (отправка в базу).
*   Удалим функции для WeekId, так как база сама генерирует ID или мы используем даты.

## 3. Обновление компонентов
Так как функции станут *асинхронными*, интерфейс должен уметь ждать загрузку (показывать спиннер/лоадер).

*   **Dashboard.tsx**:
    *   Добавить состояние `loading`.
    *   Загружать данные через `useEffect` при старте.
*   **Manage.tsx**:
    *   Добавить загрузку.
    *   Кнопка удаления будет отправлять запрос в базу.
*   **AddEmployee.tsx**:
    *   При сохранении ждать ответа от базы перед переходом назад.
